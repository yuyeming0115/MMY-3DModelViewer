# 3D 模型预览器（本地化轻量工具）规划方案

> 参考网页：https://www.thepoly.cn/tools/model-viewer
> 目标：单文件 exe、双击即用、无需安装的本地化 3D 模型预览工具

---

## 一、项目目标

开发一款**单文件 exe、双击即用、无需安装**的本地化 3D 模型预览工具，核心诉求是"轻量 + 离线 + 等效功能"，对标 thepoly.cn/tools/model-viewer 的功能。

---

## 二、功能需求

### 2.1 核心功能（对标网页）

| 模块 | 功能点 | 说明 |
|------|--------|------|
| **模型导入** | 拖拽 / 点击导入 | 支持多文件批量导入 |
| **格式支持** | FBX / GLB / glTF / OBJ / PLY | 与网页一致，可扩展 STL/3MF |
| **3D 预览** | 旋转 / 平移 / 缩放 | 鼠标 + 触控板交互 |
| **模型信息** | 顶点数 / 面数 / 材质数 / 贴图数 / 动画数 / 包围盒 | 导入后统计显示 |
| **贴图导出** | PNG / JPG / WebP | 从模型提取贴图并导出 |
| **场景控制** | 网格地面 / 坐标轴 / 背景色 / 灯光 | 辅助查看 |
| **显示模式** | 实体 / 线框 / 法线 / 骨骼 | 多模式切换 |
| **动画播放** | 播放 / 暂停 / 进度 / 循环 | 支持 FBX/glTF 动画 |

### 2.2 增强功能（本地化优势）

- **最近打开列表**：本地持久化最近 10 个文件
- **截图导出**：当前视角 PNG 截图
- **格式转换**：FBX/OBJ → GLB（离线转换，可选）

## 三、技术选型

### 3.1 方案对比

| 方案 | 体积 | 内存 | WebGL | 单文件打包 | 推荐度 |
|------|------|------|-------|-----------|--------|
| **Tauri 2.x + Three.js** | ~3-8MB | 低 | 系统WebView | 支持 | ⭐⭐⭐⭐⭐ |
| Electron + Three.js | ~80-150MB | 高 | 内置Chromium | 需打包 | ⭐⭐ |
| Python + PyOpenGL | ~30-50MB | 中 | 自渲染 | 需PyInstaller | ⭐⭐ |
| C++ + Qt + OpenGL | ~20-40MB | 低 | 自渲染 | 支持 | ⭐⭐⭐ |

### 3.2 推荐技术栈：Tauri 2.x + Vue3 + Three.js

**理由：**

1. **轻量**：Tauri 复用系统 WebView，exe 体积仅 3-8MB，符合"轻量"诉求
2. **生态**：Three.js 与网页同源，可直接复用 loader 与渲染逻辑，功能等效性最高
3. **单文件**：Tauri 2.x 支持 portable 模式，生成单 exe 无需安装
4. **开发效率**：前端 Vue3 + Three.js 开发体验好，Rust 后端处理文件 IO

**关键技术点：**

- Three.js Loaders：`FBXLoader` / `GLTFLoader` / `OBJLoader` / `PLYLoader`
- 动画：`AnimationMixer`
- 贴图提取：遍历 `material.map` 等属性导出 `Texture.image`
- 单文件打包：Tauri `portable` feature + 资源内嵌

## 五、开发阶段规划

### 阶段 1：MVP 核心预览（先跑起来）

- Tauri 项目搭建 + Vue3 + Three.js 集成
- 拖拽/点击导入 GLB/glTF（最易实现）
- 基础相机控制（OrbitControls）
- 模型信息统计显示
- **验收**：能导入 gltf 并旋转查看

### 阶段 2：多格式 + 动画

- 接入 FBXLoader / OBJLoader / PLYLoader
- AnimationMixer 动画播放控制
- 显示模式切换（实体/线框/法线/骨骼）
- 网格、坐标轴、灯光辅助
- **验收**：FBX 动画可播放，多格式可加载

### 阶段 3：贴图导出 + 截图

- 遍历材质提取贴图
- PNG/JPG/WebP 导出（Tauri 文件保存对话框）
- 当前视角截图导出
- **验收**：贴图与截图可保存到本地

### 阶段 4：体验优化 + 打包

- 最近打开列表（本地 JSON 持久化）
- 错误处理（格式不支持/解析失败的友好提示）
- 加载进度条、大模型性能优化
- Tauri portable 模式打包单 exe
- 图标、签名、体积优化
- **验收**：单 exe 双击可用，体积 < 10MB

## 六、关键技术风险与对策

| 风险 | 对策 |
|------|------|
| FBXLoader 依赖 `fflate`，体积大 | 按需加载，打包时 tree-shaking |
| 系统 WebView 差异（WebView2/WebKit） | 限定 Win10+，打包时检测 WebView2 |
| 大模型内存溢出 | 加载前预估，>100MB 警告 |
| Tauri 单文件模式限制 | 资源全部内嵌 `resources` + `dist` |
| 贴图格式多样（KTX2/DDS） | 集成 KTX2Loader，必要时转码 |
