1项目目标

开发一款单文件 exe、双击即用、无需安装的本地化 3D 模型预览工具，核心诉求是"轻量 + 离线 + 等效功能"，对标 thepoly.cn/tools/model-viewer 的功能。

参考网页：https://www.thepoly.cn/tools/model-viewer

3.2 推荐技术栈：Tauri 2.x + Vue3 + Three.js

  • 轻量：Tauri 复用系统 WebView，exe 体积仅 3-8MB
  • 生态：Three.js 与网页同源，可直接复用 loader 与渲染逻辑
  • 单文件：Tauri 2.x 支持 portable 模式，生成单 exe 无需安装
  • 开发效率：Vue3 + Three.js 开发体验好，Rust 后端处理文件 IO

关键技术点：FBXLoader / GLTFLoader / OBJLoader / PLYLoader、AnimationMixer、材质贴图提取、Tauri portable 打包

5开发阶段规划

阶段 1：MVP 核心预览 先跑起来
  • Tauri 项目搭建 + Vue3 + Three.js 集成
  • 拖拽/点击导入 GLB/glTF（最易实现）
  • 基础相机控制（OrbitControls）
  • 模型信息统计显示
验收：能导入 gltf 并旋转查看
阶段 2：多格式 + 动画 功能扩展
  • 接入 FBXLoader / OBJLoader / PLYLoader
  • AnimationMixer 动画播放控制
  • 显示模式切换（实体/线框/法线/骨骼）
  • 网格、坐标轴、灯光辅助
验收：FBX 动画可播放，多格式可加载
阶段 3：贴图导出 + 截图 导出能力
  • 遍历材质提取贴图
  • PNG/JPG/WebP 导出（Tauri 文件保存对话框）
  • 当前视角截图导出
验收：贴图与截图可保存到本地
阶段 4：体验优化 + 打包 交付
  • 最近打开列表（本地 JSON 持久化）
  • 错误处理、加载进度条、大模型性能优化
  • Tauri portable 模式打包单 exe
  • 图标、签名、体积优化
验收：单 exe 双击可用，体积 < 10MB

6关键技术风险与对策

FBXLoader 依赖 fflate，体积大
按需加载，打包时 tree-shaking
系统 WebView 差异
限定 Win10+，打包时检测 WebView2
大模型内存溢出
加载前预估，>100MB 警告
Tauri 单文件模式限制
资源全部内嵌 resources + dist
贴图格式多样（KTX2/DDS）
集成 KTX2Loader，必要时转码
